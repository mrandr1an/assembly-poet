1.1. Types

1.1.1. Constants

Nasm has four types of constants numeric , character, string and floating-point.

  1. Numeric

    Numeric constants as expected are just numbers. Here are some ways that you can represent different bases according to the docs.

    Base Prefix Number in nasm Number in Base 10
    10 0d 0d200 200
    16 0x 0xC8 200
    8 0o 0o310 200
    2 0b 0b11001000 200

    This is the way I prefer to write numbers as anyways. It really should not matter and you can always refer to the docs above to find more ways of representing a value.

  2. Character Strings

    Character strings consist of up to 8 characters in quotes ("). When using (`) they support C-Style escapes for special characters (ex. \n).

  3. Character Constants

    Essentially are integers, are up to 8 bytes long and used in an expression context only, 

    mov eax, 'hello'

    Note, a character with more than one byte will be arranged with little endian order in mind. So the above is equal to 

    mov eax, 0x6F6C6C6568

    So, essentially its ASCII -> Numerical Representation -> Reverse.

  4. String Constants

    Character Strings that are used in pseudo-instructions like the DB family of instructions and INCBIN.

  5. Unicode Strings

    Using some simple operators, ?utf16?, ?utf16le?, ?utf16be?, ?utf32?, ?utf32le? and ?utf32be? you can easilly create unicode formatted strings. Here is the example from the docs: 

    %define u(x) __?utf16?__(x) 
%define w(x) __?utf32?__(x) 
      dw u('C:\WINDOWS'), 0       ; Pathname in UTF-16 
      dd w(`A + B = \u206a`), 0   ; String in UTF-32
  6. Floating Point Constants

    Acceptable only as arguments to DB, DW, DD, DQ, DT, and DO, or as arguments to the special operators ?float8?__, ?float16?, ?bfloat16?,__?float32?__, ?float64?, ?float80m?, ?float80e?, ?float128l?, and ?float128h?.

1.1.2. Expressions

According to the docs expressions are evaluated as 65-bit integers which are then adjusted to the appropriate size. NASM supports two special tokens $ which evaluates to the position at the start of the line that contains the expression and \[ which evaluates to the biginning of the section the current expression is in. *You can tell how far into the section you are by using ($-\])*.
